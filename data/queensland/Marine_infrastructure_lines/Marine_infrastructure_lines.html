Description

Abstract / Description:
This topographic dataset displays long narrow structures associated with the marine environment (e.g., breakwaters, jetties, sea walls) built along Queensland's coastline or inland waterbodies
Purpose:
This Queensland wide dataset provides generalised information for topographical mapping purposes. To provide digital data displaying long narrow structures associated with the marine environment (e.g., breakwaters, jetties, sea walls) built along Queensland's coastline or inland waterbodies. The primary objective of this dataset is to provide a cartographic portrayal of Queensland's constructed features and should be interpreted and utilised correctly. It is suitable for use in land administration, emergency management, topographic mapping and in the production of web based mapping applications.
Additional Information:
Names have been sourced from numerous locations and attributes within the data describe the source of the names.

- Queensland Place Names Database
- Land parcel boundaries Queensland

Feature Types:
BOAT RAMP – A sloping construction to facilitate launching or retrieving vessels from water
BREAKWATER – A structure built to break the force of waves to protect a harbour, river mouth or other waterfront facility.
DRY DOCK – An artificial basin fitted with gate or caisson into which a vessel may be floated and from which the water may be pumped out to expose the bottom of the vessel.
GROYNE – A low artificial wall-like structure constructed of timber, stone, concrete, or steel, usually extending roughly perpendicular to the shoreline or coastline, and designed to protect the shore from erosion or to trap sand.
JETTY – A public or commercial structure extending into the water to afford a berthing place for vessels or to serve as a pleasure promenade.
SEAWALL – A solid structure usually of concrete masonry or earth, built to prevent erosion or encroachment by the sea.
WHARF LINE – A narrow structure built from the land parallel to shore to provide for the berthing of vessels.
OTHER MARINE INFRASTRUCTURE LINE – A long, narrow structure associated with the marine environment and not covered by any of the above Attributes:

FEATURE_TYPE (Feature Type) - Unique feature type that identifies the type of feature. (options: Boat Ramp, Breakwater, Dry Dock, Groyne, Jetty, Sea Wall, Wharf Line, Other Marine Infrastructure Line)
NAME (Name) - The name of the feature (if available)
ALTERNATE_NAME (Alternate Name) - An alternate name of the feature if available from the Queensland Place Names Database, otherwise left blank
QLD_PNDB_ID (Qld PNDB ID) - Queensland Place Names Database unique identifier
ATTRIBUTE_SOURCE (Attribute Source) - The source imagery, map, or data used for attributing the feature. If the feature is named, the attribution would be the source information that the name was obtained from. If the feature is not named, the attribution is the source information which defines what the feature is e.g. breakwater
ATTRIBUTE_DATE (Attribute Date) - Date of the source imagery, map, or data used for attributing the feature.
ADDITIONAL_NAMES (Additional names) - Other names of the feature. Maybe commonly known by other names or have different spelling. Multiple names can be shown, separated by a comma
ADD_NAMES_SOURCE (Additional Names Source) - The source map or data that the additional names were obtained from. Multiple sources can be shown, separated by a comma. If multiple sources are shown, the written sequence will be the same as the written sequence of the names themselves.
FEATURE_SOURCE (Feature Source) - The source imagery, map, or data that the spatial location of the feature was obtained from.
FEATURE_DATE (Feature Date) - Date of the source imagery, map, or data that the spatial location of the feature was obtained from.
PFI (PFI) - A Persistent Feature Identifier (PFI) is generated once for each feature at the point of creation. The value of the PFI will stay with the feature through all changes to the feature (both spatial and non-spatial) until the feature is retired.
UFI (UFI) - A Unique Feature Identifier (UFI) is generated once for each new feature at the point of creation (at this point the PFI and UFI will be the same). The value of the UFI will stay with the feature through all changes to the feature (both spatial and non-spatial) unless the feature is split into multiple other parts. If the feature is split, the separate parts will retain the original PFI but new UFI’s will be generated for the split parts.
DIMENSION_M (Dimension (m)) - The length of the feature measured in square metres (Albers projection)
UPPER_SCALE (Upper Scale) - The upper scale for which the feature should be considered suitable
TEXT_NOTE (Text Note) - For mapping purposes. Additional annotation that can be shown on a map.
ADD_INFORMATION (Additional Information) - A comment field (additional information)

History:

Lineage statement:
Initially for whole of state data capture a point is positioned in the centre of the feature, The location of features was sourced from government or community website and validated against departmental imagery (orthophotography and satellite). Other data was supplied by Geoscience Australia as part of the National Topographic Information capture initiative or from existing departmental regional mapping or topographic datasets.

This dataset is GDA2020 Compliant. The source data, and therefore the captured features, were captured as GDA94 (horizontal only). This data has been projected to GDA2020 by using ArcGIS Pro geoprocessing tool Project geoprocessing. Using geographic transformation parameters ‘GDA94 to GDA2020 NTv2 3 Conformal’.

The data has also been processed to remove any redundant vertices by using ArcGIS Pro’s Simplify Line geoprocessing tool. Retain critical points (Douglas-Peucker) method was used for the Simplification Algorithm and a Simplification tolerance of 0.25m. The data was further processed using an ESRI validation in a Repair Geometry geoprocessing tool and deleting any features with Null geometry.
